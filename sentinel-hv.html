Purpose

Use this page as the presentation shell around the repo. It is intentionally separated from the existing scanner so you can evolve the design, swap modules, and paste in your own implementation notes without disturbing the current shell scripts.

Software Flow

A more structured flow that turns the project into a staged system rather than a single scan pass.

Phase 0

Deployment

  • Package a local agent or module set.
  • Define invocation parameters and target scope.
  • Record launch metadata and session identifiers.
Phase 1

Profiling

  • Collect platform indicators, virtualization artifacts, and environment signals.
  • Normalize findings into a target profile.
  • Emit `target_profile.json` or equivalent metadata.
Phase 2

Scanning

  • Run structured checks by version, capability, and configuration.
  • Associate findings with a local ruleset or advisory dataset.
  • Emit a normalized report payload for downstream analysis.
Phase 3

Decision Engine

  • Correlate evidence across platform, service, and policy layers.
  • Apply classification rules and confidence scoring.
  • Select follow-on modules based on the evidence state.
Phase 4

Reporting and Output

  • Generate human-readable output, machine-readable JSON, and module summaries.
  • Store artifacts in a predictable local structure.
  • Preserve enough context for future amendments, removals, or replacements.

Module Map

These module blocks give your future code a clean place to land without forcing changes into the current Bash entrypoints.

Profiler Module

Responsible for environment fingerprints, virtualization indicators, OS context, and host-guest surface mapping.

Scanner Module

Runs the main checking engine and records structured findings keyed by target, capability, and evidence source.

Correlation Module

Scores findings, resolves conflicts, groups evidence, and determines the most likely target role or platform state.

Reporting Module

Builds terminal summaries, JSON artifacts, and future HTML or markdown outputs from the normalized result object.
